3. Recommended tech stack

Framework

Next.js 15 (App Router)
Why not Astro: Your games need client-side interactivity (map interactions, timers, drag/sort). Astro is great for static content sites but you'd need to hydrate heavy island components anyway. Next.js gives you SSR for the hub/shell, client components for games, ISR for daily content rotation, and API routes for score validation — all in one framework.

Why not plain HTML: You have 5+ games sharing common patterns (scoring, streaks, sharing, analytics, auth). A component framework eliminates the 80% code duplication you currently have across 10+ HTML files.

UI / components

React 19 + Tailwind CSS 4
Shared game shell: Header, score display, streak tracker, share button, clue bar — these are reusable React components consumed by every game renderer.

Tailwind: Matches your prototype's utility-first approach. Fast to iterate, consistent spacing/colour, responsive by default. No CSS duplication across game files.

Maps

MapLibre GL JS (replaces Leaflet)
Why switch: Leaflet is 180KB and renders on the DOM (slow for complex GeoJSON). MapLibre GL is WebGL-rendered, handles your suburb polygons more smoothly, supports vector tiles (smaller payloads than raw GeoJSON), and is open source. 3 of your 5 games use maps — this is a significant performance win.

Alternative: If you want to keep Leaflet for simplicity, at minimum self-host it and precache in the service worker. But MapLibre is the modern choice.

Offline + PWA

Workbox 7 + next-pwa
Cache strategies:
— App shell (HTML/CSS/JS): Cache-first, update on new deploy
— Game JSON (today's config): Stale-while-revalidate
— GeoJSON suburb data: Cache-first (rarely changes)
— Archive images: Cache-first with size limit
— API calls (scores, leaderboard): Network-first with offline queue

Pre-fetch: Service worker fetches tomorrow's game JSON overnight. User opens app on the bus — instant load.

Local storage

idb-keyval (IndexedDB wrapper) — 600 bytes
What gets stored locally:
— Player streaks (per game)
— Current game progress (survive page refresh)
— Completed games today (don't replay)
— User preferences (region, notification opt-in)
— Offline event queue (flush when online)

Why not localStorage: 5MB limit, synchronous (blocks main thread), no structured data. IndexedDB handles all of this better. idb-keyval makes it as simple as get/set.

CMS + content pipeline

Sanity.io (hosted) or Strapi 5 (self-hosted)
Sanity advantages: Real-time collaboration, structured content schemas, built-in image pipeline (replaces some Transform work), webhook on publish, generous free tier (3 users, 500K API requests/mo).

Editorial workflow: AI agent submits draft → editor reviews in Sanity Studio → schedules for 5am AEDT → webhook triggers Next.js ISR rebuild → CDN purges stale content.

If self-hosting required: Strapi 5 on an ACM-managed server. Same schema approach, more ops overhead.

AI content agents

Cloud Functions (GCP/AWS Lambda) + Claude API via Sochi
Agent pipeline:
1. Cron triggers agent function daily/weekly
2. Agent queries Silverstone (archive stories) + Valencia (photos)
3. Agent calls Claude API via Sochi proxy with structured output schema
4. Agent validates JSON against game-type schema
5. Agent submits to Sanity as draft via API
6. Editor reviews and publishes

Per-game agent: Each game type has its own prompt template and validation rules. Trivia agent generates questions from archive content. Flashback agent pairs photos with year + fact. Crossword agent generates grids with local clue themes.

Hosting + edge

Vercel (primary) or Cloudflare Pages
Vercel: Zero-config Next.js deployment, edge functions for score validation, built-in ISR, preview deployments for editorial review, Web Analytics included.

Cloudflare alternative: If ACM prefers to keep infrastructure in-house, Cloudflare Pages + Workers + KV gives you the same edge-first architecture with more control.

Subdomain: games.canberratimes.com.au → Vercel/CF. Shares .canberratimes.com.au cookie domain for Piano session passthrough.

Analytics

PostHog (self-hosted or cloud)
Why PostHog over Amplitude: Open source, can self-host (ACM data sovereignty), built-in session replay (debug UX issues), feature flags (A/B test game variants), funnel analysis, and cohort retention — all in one tool. Free tier: 1M events/mo.

Critical: Every event carries piano_uid as a user property. This is joined with Dijon subscription data for the retention correlation analysis.

Push notifications

Firebase Cloud Messaging (FCM) via Web Push API
Daily puzzle notification: "Your daily games are ready 🧩" at 5am AEDT. Simple, proven, free. FCM handles the push infrastructure; your service worker displays the notification.

PWA install prompt: After 3 consecutive daily visits, prompt "Add to Home Screen" for app-like experience. Installed PWA = higher retention.

7. Accessibility (a11y) — WCAG 2.1 AA from day 1

Non-negotiable. Your audience is Canberra — 70%+ public servants. Government agencies mandate WCAG 2.1 AA compliance for digital products they engage with. ACM as a publisher also has obligations under the Disability Discrimination Act 1992. Building a11y in from the start costs 10% of what retrofitting later does.

Keyboard navigation

Every game fully playable without a mouse
Crossword: Arrow keys to move between cells, Tab to jump to next word, Shift+Tab for previous, Enter to toggle across/down. Your prototype uses a hidden input hack for mobile — for desktop, implement proper keydown handlers on the grid with visible focus indicators.

Map games: This is the hardest one. Map polygons need to be focusable via Tab. Add an alternative text input: type suburb name instead of clicking. Dual-mode: visual click OR keyboard search.

North vs South: Already good — two buttons, arrow keys or 1/2 keys can map to North/South.

Flashback Friday: Slider needs arrow key increment (already native for range input), Enter to submit.

Screen reader support

ARIA labels, live regions, semantic HTML
ARIA live regions: Score changes, timer countdowns, feedback messages ("Correct! +3 points") must be announced via aria-live="polite". Without this, screen reader users get no feedback on their actions.

ARIA labels: Every interactive element needs a descriptive label. Map polygons: aria-label="Suburb: Braddon". Crossword cells: aria-label="Row 1, Column 3, 1 Across". Buttons: already labelled in your prototypes (good).

Game state announcements: "Round 3 of 10. Find: Telstra Tower. 8 seconds remaining." Announced on round start via an off-screen live region.

Colour-blind safe design

Never use colour alone to communicate state
Current problem: Your prototypes use green = correct, red = wrong, orange = missed. ~8% of males are red-green colour blind. These states are indistinguishable.

Fix: Add a secondary indicator alongside colour. Correct: green fill + checkmark icon. Wrong: red fill + X icon. Missed: orange fill + "?" icon. On the crossword: correct cell gets a subtle checkmark overlay, not just a colour change.

Implementation: Use pattern fills or icons. Add a "colour-blind mode" toggle in settings that switches to high-contrast patterns (stripes, dots, crosshatch) instead of relying on colour ramps. Store preference in IndexedDB.

Motion and timing

Respect prefers-reduced-motion, adjustable timers
Reduced motion: Your confetti celebrations, shake animations, and map fly-to transitions should be wrapped in @media (prefers-reduced-motion: no-preference). Users with vestibular disorders can disable them.

Timed games: Hunt the Landmark (8 sec) and North vs South (5 sec) are accessibility barriers for users with motor or cognitive disabilities. Add an "Extended time" mode (2x timer) in settings. This doesn't affect scoring fairness — store it as a separate leaderboard category or simply don't penalise time in accessibility mode.

Auto-advance: Your 3-second auto-advance between rounds (setTimeout) should be optional. Add a "Next" button that also works, so users can control the pace.

8. Dark mode — built into the design system, not bolted on

Why Phase 1: Your games are played on the 7am bus to Civic, in bed before sleep, and during boring meetings. All low-light scenarios. Dark mode isn't a preference — it's when most of your engagement will happen. NYT shipped it across the entire Games app. Build it in from day 1 or you'll retrofit every component later.

Implementation approach

Tailwind dark: variant + CSS custom properties
Tailwind 4: Use dark: variants throughout. className="bg-white dark:bg-gray-900". This is zero-cost at build time — Tailwind only generates classes you use.

Each game type has dark mode nuances

9. Hints system — reduce abandonment, not difficulty

Why this matters: A player who gets stuck and abandons a puzzle has a broken streak, a bad experience, and a higher churn probability. A player who uses a hint and completes the puzzle has an intact streak, a small dopamine hit from completion, and returns tomorrow. NYT offers hints across Spelling Bee, Crossword, and Connections. Hints protect retention at the cost of a slightly easier game — that's a trade worth making.

Crossword hints

3 tiers of progressive assistance
Tier 1 — Check cell: Tap "Check" on a single cell. If wrong, it flashes red and clears. If correct, it gets a subtle lock icon. Player still needs to find the right letter. Cost: free, unlimited.

Tier 2 — Reveal letter: Reveals one correct letter in the active word. Cost: 1 hint token per use. Score penalty: -50 points per reveal.

Tier 3 — Reveal word: Fills in the entire active word. Cost: 2 hint tokens. Score penalty: word doesn't count toward "no hints" badge.

Hint tokens: Players get 3 free hint tokens per day. Resets at midnight. Premium subscribers get 5. This creates a resource management layer — do you spend your hint on 1-Across or save it for the tricky down clue?

Hunt the Landmark hints

Progressive map narrowing
Tier 1 — Zoom hint: Map auto-zooms to the correct district (e.g., Inner North). Reduces the search area by ~80%. Cost: 1 hint token. Timer pauses during hint animation.

Tier 2 — Distance ring: Shows a circle (2km radius) centered on the correct location. Cost: 2 hint tokens. Significantly narrows the target but still requires precision.

Implementation: Both hints are just Leaflet/MapLibre operations — map.flyTo() and L.circle(). No server-side changes needed. Hint usage tracked in analytics for difficulty calibration.

Map games + North vs South hints

Eliminate wrong answers, brief reveal
Map games — Neighbourhood hint: Highlights the 2-3 suburbs adjacent to the target. "It's near one of these." Cost: 1 hint token. Reduces options without giving the answer.

North vs South — Brief flash: The target suburb flashes on the map for 1 second before disappearing. Player sees the location, then answers. Cost: 1 hint token. Timer pauses during flash.

North vs South — 50/50: Not applicable (already binary). Instead, add a "Skip" that doesn't break the streak but scores 0. Already exists in your prototype.

Flashback Friday hints

Decade bracket + contextual clue
Tier 1 — Decade hint: Narrows the slider range to the correct decade (e.g., "This photo is from the 1990s"). Slider min/max updates visually. Cost: 1 hint token.

Tier 2 — Context clue: Reveals a text hint from the CMS: "This was taken during a sporting event." Doesn't give the year but provides context. Cost: 1 hint token.

CMS schema addition: Each photo config gets a hint_decade (auto-derived from year) and hint_context (written by editor/AI). Example: { "hint_decade": "1990s", "hint_context": "Taken at a Grand Final" }
